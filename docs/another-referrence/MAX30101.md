# MAX30101 传感器底层驱动开发参考（补充完整版）

本文档聚焦 MAX30101 在 MCU 侧的底层驱动实现，适合用于初始化、寄存器配置和 FIFO 采样解析的快速对照。

## 1. 硬件接口与电源规范

### 1.1 通信接口
支持协议: I2C 两线串行接口。
I2C 最大时钟: `400 kHz`。
7-bit 从机地址: `0x57`（固定 `1010111`）。
8-bit 写地址（`MAX30101_I2C_ADDR_WRITE`）: `0xAE`。
8-bit 读地址（`MAX30101_I2C_ADDR_READ`）: `0xAF`。

### 1.2 硬件连接要点
SDA 为漏极开路；SDA、SCL 均需外部上拉电阻（通常 > `500 ohm`）。
INT 为低电平有效、漏极开路输出，建议外接 `4.7k ohm` 上拉至最高 `+5V`。
建议在电源引脚附近布置旁路电容，抑制 LED 脉冲导致的电源纹波。

### 1.3 上电时序
推荐先上 `VDD (1.8V)`，再上 `VLED+ (5.0V)`。
上电后会触发 `PWR_RDY` 中断，表示芯片可进入初始化。

### 1.4 I2C通信的DMA实现细节
MAX30101的I2C通信可通过DMA实现，以减少CPU占用。DMA配置时需注意以下要点：
- DMA传输应配置为每次传输9字节（3通道×3字节），因为每个样本包含9字节数据。
- 传输完成后，DMA会自动更新I2C地址指针，允许连续读取FIFO数据。
- 为了防止FIFO写满，DMA传输频率应略高于FIFO填充率（例如800Hz采样率，DMA传输频率可设置为1kHz）。
- 在DMA传输过程中，应确保FIFO中有足够数据（例如FIFO_A_FULL=15），避免传输时FIFO已满。
- 由于FIFO_DATA寄存器的特殊性（读取时地址指针不自动递增），DMA传输应设置为连续读取，而非单次读取。

## 2. 核心寄存器与位定义

### 2.1 中断与状态寄存器
读取状态寄存器 `0x00` 或 `0x01` 会清除对应硬件中断标志，并释放 INT 低电平。

```c
/* INT_STAT_1 (0x00) */
#define MAX30101_INT_STAT_1_REG 0x00
// [Bit 7] A_FULL : FIFO 几乎满标志。1 = FIFO 剩余空间达到设定阈值；0 = 正常。
// [Bit 6] PPG_RDY : 新数据就绪标志。1 = FIFO 中有新的采样数据；0 = 无新数据。
// [Bit 5] ALC_OVF : 环境光消除溢出。1 = 环境光过强导致 ADC 饱和；0 = 正常。
// [Bit 0] PWR_RDY : 电源就绪标志。1 = VDD 上电越过 UVLO 阈值，可开始初始化（不可屏蔽）。

/* INT_STAT_2 (0x01) */
#define MAX30101_INT_STAT_2_REG 0x01
// [Bit 1] DIE_TEMP_RDY : 内部温度转换完成。1 = 温度结果可读；0 = 正在转换或未触发。
```

### 2.2 FIFO 相关寄存器
```c
/* FIFO pointers & data */
#define MAX30101_FIFO_WR_PTR_REG 0x04 // [4:0] 写指针，硬件在新样本写入 FIFO 后自动自增。
#define MAX30101_OVF_COUNTER_REG 0x05 // [4:0] 溢出计数，最大饱和值 0x1F；表示未及时读取导致的数据覆盖情况。
#define MAX30101_FIFO_RD_PTR_REG 0x06 // [4:0] 读指针，I2C 读取 FIFO 数据后硬件自动自增。
#define MAX30101_FIFO_DATA_REG 0x07 // FIFO 数据读取窗口；连续读取该地址即可流式取数。

/* FIFO_CONFIG (0x08) */
#define MAX30101_FIFO_CONFIG_REG 0x08
// [7:5] SMP_AVE : 硬件采样平均设置（降低噪声与数据吞吐）。
// 000 = 1 次平均，001 = 2 次，010 = 4 次，011 = 8 次，100 = 16 次，101 = 32 次，110 = 32 次，111 = 32 次。
// [4] FIFO_ROLLOVER_EN : FIFO 满载覆盖使能。1 = 继续写并覆盖旧数据；0 = 满后停止写入。
// [3:0] FIFO_A_FULL : FIFO 几乎满中断阈值。用于设定剩余空位到达多少时触发 A_FULL。
// 0x0 = 0个样本剩余，0x1 = 1个样本剩余，...，0xF = 15个样本剩余。
```

FIFO 读写指针回绕处理注意事项
由于 FIFO 深度为 32（指针范围 0~31），读写指针在递增至 31 后会自动回绕到 0。计算 FIFO 中可用样本数时，必须考虑回绕情况。基本公式为：`可用样本数 = (WR_PTR + 32 - RD_PTR) % 32`。若未正确处理回绕，在指针跨越边界时可能计算出负值或错误数值。建议在每次读取前重新读取指针并计算，避免使用缓存值。

此外，清空 FIFO 时应将 WR_PTR、RD_PTR 和 OVF_COUNTER 全部写 0，确保指针状态已知。

### 2.3 模式与采样配置寄存器
```c
/* MODE_CONFIG (0x09) */
#define MAX30101_MODE_CONFIG_REG 0x09
// [7] SHDN : 休眠控制位。1 = 休眠模式；0 = 活动模式。
// [6] RESET : 软复位控制位。1 = 触发复位；0 = 正常工作。
// [2:0] MODE : 工作模式选择。
// 010 = HR mode (仅 RED)
// 011 = SpO2 mode (RED + IR)
// 111 = Multi-LED mode (GREEN/RED/IR 可按时隙组合)
// 说明: 该寄存器还包含 RESET/SHDN 等系统级控制位，驱动初始化通常会先用 RESET 软复位。

/* SPO2_CONFIG (0x0A) */
#define MAX30101_SPO2_CONFIG_REG 0x0A
// [6:5] SPO2_ADC_RGE : ADC 满量程范围控制。
// 00 = 2048nA，01 = 4096nA，10 = 8192nA，11 = 16384nA。
// [4:2] SPO2_SR : ADC 采样率（50 ~ 3200 sps）。
// 000 = 50 sps，001 = 100 sps，010 = 200 sps，011 = 400 sps，100 = 800 sps，101 = 1000 sps，110 = 1600 sps，111 = 3200 sps。
// [1:0] LED_PW : LED 脉宽/ADC 分辨率耦合设置。
// 00 = 69us (15-bit)，01 = 118us (16-bit)，10 = 215us (17-bit)，11 = 411us (18-bit)。
```

采样率寄存器值的具体设置（SPO2_CONFIG 组合）
写入 `SPO2_CONFIG` 时需要将 ADC 量程、采样率和脉宽三个字段组合成一个字节。各字段的位偏移已在寄存器定义中说明。

例如，配置为 ADC 量程 8192nA（编码 `10`）、采样率 800sps（编码 `100`）、脉宽 215µs（编码 `10`）时，组合值为：
`(0b10 << 5) | (0b100 << 2) | 0b10 = 0x40 | 0x10 | 0x02 = 0x52`。

若直接按表格中的数值写入而不进行移位，会导致配置错误。建议使用宏或函数来组合字段。

### 2.4 LED 电流与多时隙控制
```c
/* LED current registers */
#define MAX30101_LED1_PA_REG 0x0C // RED
#define MAX30101_LED2_PA_REG 0x0D // IR
#define MAX30101_LED3_PA_REG 0x0E // GREEN

/* Multi-LED slot control */
#define MAX30101_MULTI_LED_CTRL1 0x11
// [6:4] SLOT2, [2:0] SLOT1
// 槽位编码: 001=LED1_RED, 010=LED2_IR, 011=LED3_GREEN, 000=Disabled。

#define MAX30101_MULTI_LED_CTRL2 0x12
// [6:4] SLOT4, [2:0] SLOT3
// 多光路输出时，一个完整样本按 SLOT1->SLOT2->SLOT3->SLOT4 顺序压入 FIFO。
```

LED 电流估算`I_LED ~= RegValue * 0.2 mA`。
例如，LED1_PA = 0x3F，对应电流约 12.6mA。

LED 电流与寄存器值对应关系
| LEDx_PA[7:0] | TYPICAL LED CURRENT (mA) |
|--------------|------------------------|
| 0x00h        | 0.0                    |
| 0x01h        | 0.2                    |
| 0x02h        | 0.4                    |
| ...          | ...                    |
| 0x0Fh        | 3.0                    |
| ...          | ...                    |
| 0x1Fh        | 6.2                    |
| ...          | ...                    |
| 0x3Fh        | 12.6                   |
| ...          | ...                    |
| 0x7Fh        | 25.4                   |
| ...          | ...                    |
| 0xFFh        | 51.0                   |

### LED电流选择的推荐值
LED电流选择对PPG信号质量和功耗有重要影响，推荐值如下：
- 低功耗应用（如手表）：8-10mA (0x28-0x2E)
- 通用应用（如健身手环）：12-15mA (0x3F-0x48)
- 高精度应用（如医疗级设备）：15-20mA (0x48-0x64)
- 注意：LED电流超过51mA(0xFF)可能导致LED过热，应避免使用。

### 2.5 环境光干扰处理
当ALC_OVF中断触发时，表示环境光过强导致ADC饱和，需要采取措施：
- 首先，检查ALC_OVF中断标志，确认环境光干扰。
- 可能的处理措施：
  1. 减小LED电流（例如从12.6mA减至8mA）
  2. 增加环境光屏蔽，例如使用遮光罩
  3. 调整LED脉宽（如从215μs增加到411μs）
  4. 降低采样率（如从800sps降至400sps）
- 通常，优先尝试减小LED电流，因为这是最直接的解决方案。

## 3. 目标配置示例: 三光路 + 100Hz 输出 + 8 次平均 (定时器异步轮询模式)

- **目标**: `GREEN + RED + IR` 轮切，输出数据率（ODR）要求严格对齐 MCU 内部的 `100 Hz`，硬件平均 `8x`。由于硬件上未连接 INT 引脚，必须由 MCU 的 10ms 主定时器异步驱动 I2C 读取。
- **ADC 采样率关系**: 内部实际采样率 $f_{ADC} = f_{ODR} \times N_{AVG} = 100 \times 8 = 800 Hz$。因此 `SPO2_SR` 配置为 `800 sps`（编码 `100`）。
- **脉宽限制**: 在 `800 sps` 且启用 3 个 LED 通道的情况下，`LED_PW` 最大只支持 `215 us`，推荐 `LED_PW = 0x02`（17-bit 分辨率）。
- **时隙配置 (Slots)**: `0x11 = 0x13`（SLOT1=GREEN(011), SLOT2=RED(001)）。`0x12 = 0x02`（SLOT3=IR(010), SLOT4=Disabled(000)）。
- **FIFO 帧载荷**: 3 个光路，每路 3 字节 -> `1 sample = 9 bytes`。在 100Hz 轮询读取下，每次 10ms 中断通常只会产生 1 个 sample，即单次 DMA 突发读取 `9 bytes`。

## 4. 标准数据采集流程 (MCU 定时器驱动 I2C+DMA 非阻塞流)

由于未连接硬件 EXTI 中断引脚，PPG 数据的同步与获取将完全挂载在 MCU 的主时钟（如 10ms 定时器中断）上，采用**状态机非阻塞轮询**的方式。

### 4.1 设备验证与软复位 (系统上电阶段)
- 读取寄存器 `0xFF` (Part ID)，确认返回值为 `0x15`。
- 将 `MODE_CONFIG (0x09)` 寄存器的 `RESET` 位 [6] 置 1 触发软复位，并轮询等待该位硬件自动清零。

### 4.2 清空 FIFO 状态
- 将 `FIFO_WR_PTR (0x04)`、`FIFO_RD_PTR (0x06)`、`OVF_COUNTER (0x05)` 全部写 `0x00`。
- *说明*：初始化清零可确保读写指针处于已知原点。

### 4.3 下发工作参数 (无中断配置)
- **配置 FIFO_CONFIG (0x08)**：`SMP_AVE=8` (0b011)，`FIFO_ROLLOVER_EN=1` (满载覆盖使能)。由于我们采用主动轮询，无需触发 A_FULL 中断，`FIFO_A_FULL` 可设为任意值（如 0x0F）。
  `FIFO_CONFIG = (0b011 << 5) | (1 << 4) | 0x0F = 0x7F`
- **配置 SPO2_CONFIG (0x0A)**：`SPO2_SR=800 sps` (0b100)，`LED_PW=215 us` (0b10)，`SPO2_ADC_RGE=8192nA` (0b10)。
  `SPO2_CONFIG = (0b10 << 5) | (0b100 << 2) | 0b10 = 0x52`
- **配置 LED 电流 (0x0C, 0x0D, 0x0E)**：根据透光率需求设置，例如写 `0x3F` (约 12.6mA)。
- **配置 多光路时隙 (0x11, 0x12)**：
  `MULTI_LED_CTRL1 = 0x31` (这里修正时隙编码：SLOT1=RED(001), SLOT2=GREEN(011)？根据前文需求为 GREEN->RED，则为 `0x13` 即可，即 SLOT1=011(Green), SLOT2=001(Red))。
  `MULTI_LED_CTRL2 = 0x02` (SLOT3=010(IR), SLOT4=000(Disabled))。
- **启动模式 (0x09)**：设置 `MODE_CONFIG = 0x07` 进入 Multi-LED 工作模式。**无需使能 INT_EN 寄存器**。

### 4.4 运行时非阻塞读取状态机 (核心)
此流程需在 MCU 中拆分为两个异步阶段，严禁在中断中死等 I2C 通信：

- **阶段 1：发起指针查询 (定时器 10ms 中断内触发)**
  利用 MCU 100Hz 定时器中断（提供绝对全局时间戳），调用非阻塞 I2C 接口（`HAL_I2C_Mem_Read_IT` 或 DMA），连续读取 `0x04` 和 `0x06` 寄存器获取 `WR_PTR` 与 `RD_PTR`。此时退出定时器中断。

- **阶段 2：解析指针并发起 FIFO 数据 DMA 读取 (I2C 指针读取完成回调内触发)**
  在 I2C_RxCpltCallback 中：
  1. 计算 `可用样本数 = (WR_PTR + 32 - RD_PTR) % 32`。
  2. 若 `可用样本数 > 0`，调用 `HAL_I2C_Mem_Read_DMA` 发起对 `0x07 (FIFO_DATA)` 的突发读取，长度为 `可用样本数 * 9` 字节。若为 0，说明存在时钟漂移，本次跳过。

- **阶段 3：FIFO 数据解包与入队 (I2C FIFO 数据读取完成回调内触发)**
  1. 根据 DMA 目的内存中的 9 字节数据，按每 3 字节拼接为一个 32 位无符号整数。
  2. 结合本例 `LED_PW = 10` (17-bit)，有效数据占据 Bit[17:1]，将拼接后的 32 位整型**统一右移 1 位**得到真实 ADC 值。
  3. 将解析出的 Green、Red、IR 三个变量，连同**阶段1**中打上的定时器时间戳，压入全局 Ring Buffer，完成一帧数据的无缝拼接。

## 5. 实现建议

- 先做最小可用链路：读 ID、复位、单次 FIFO 读取，再引入中断与连续流。
- 采集异常优先排查：上拉、电源纹波、LED 电流过大、FIFO 阈值设置不合理。
- 建议把寄存器配置封装为结构体，便于快速切换 `HR/SpO2/Multi-LED` 配置档。
- 调试时可先验证指针回绕逻辑：手动写入不同的 WR_PTR 和 RD_PTR 测试计算函数。
- 温度传感器虽非必需，但如需高精度 SpO2，可定期读取温度（置位 `TEMP_EN`，等待 `DIE_TEMP_RDY` 中断，读取 `0x1F`/`0x20` 并计算 `温度 = 整数 + 小数×0.0625`）。
