# LSM9DS1 传感器底层驱动开发核心规范 (6-Axis 纯净版 - 100Hz 主动查询架构)

本文档针对需要严格全局时间戳对齐的多传感器并发场景。系统主频定为 100Hz。LSM9DS1 内部采样率配置为 119Hz，**彻底关闭内部 FIFO 与硬件中断**，改由 STM32 的主定时器每 10ms 主动发起一次 SPI DMA 轮询读取。利用 BDU（防撕裂机制）与宏观降采样，完美获取时间戳严格对齐的最新横截面数据。

## 1. 通信基础接口 (Communication Interface)
* **通信总线**: SPI1 (SCK: PA5, MISO: PB4, MOSI: PB5)
* **片选引脚**: CS_A/G 连接至 PD7 (低电平有效)
* **SPI 通信参数**: 
  * 最大时钟频率: 10 MHz。
  * 时钟极性与相位: CPOL=1, CPHA=1 (SPI Mode 3)。
* **寄存器寻址机制**: A/G 传感器命令字节 **最高位 (Bit 7)** 为 R/W (0:Write, 1:Read)。突发读取时，若开启 `IF_ADD_INC`，地址会自动递增并跳过无用区域。

## 2. 核心寄存器全映射字典 (Comprehensive Register Map)

以下包含了 LSM9DS1 (A/G 部分) 所有关键寄存器的位域 (Bitfield) 说明。带 **[★核心]** 标志的为本架构中被主动配置的寄存器，其余为保留默认或未使用的寄存器（供后续功能拓展参考）。

### 2.1 身份与中断路由配置区
* **WHO_AM_I (0x0F)**: 只读，固定返回 `0x68`。用于 I2C/SPI 通信测试。
* **[★核心] INT1_CTRL (0x0C)**: INT1_A/G 引脚中断控制寄存器。
  * `[7] INT1_IG_G`: 陀螺仪中断使能。
  * `[6] INT1_IG_XL`: 加速度计中断使能。
  * `[5] INT1_FSS5`: FIFO 满中断使能。
  * `[4] INT1_OVR`: FIFO 溢出中断使能。
  * `[3] INT1_FTH`: FIFO 阈值中断使能。
  * `[2] INT1_Boot`: Boot 状态中断使能。
  * `[1] INT1_DRDY_G`: 陀螺仪数据就绪中断使能。
  * `[0] INT1_DRDY_XL`: 加速度计数据就绪中断使能。
  * **▶ 本架构配置**: `0x00` (完全禁用中断，释放 CPU 与引脚)。
* **INT2_CTRL (0x0D)**: INT2_A/G 引脚中断控制（位域定义与 INT1 类似，未引出/未使用，默认 `0x00`）。

### 2.2 陀螺仪控制寄存器区 (Gyroscope Control)
* **[★核心] CTRL_REG1_G (0x10)**: 陀螺仪基础配置。
  * `[7:5] ODR_G[2:0]`: 采样率。`011`=119Hz。
  * `[4:3] FS_G[1:0]`: 量程。`00`=245dps, `01`=500dps, `11`=2000dps。
  * `[2] 保留`: 必须为 0。
  * `[1:0] BW_G[1:0]`: 带宽。在 ODR=119Hz 时，`00`=14Hz, `01`=31Hz。
  * **▶ 本架构配置**: `0x68` (119Hz, ±500dps, 带宽 14Hz)。
* **CTRL_REG2_G (0x11)**: 陀螺仪中断与输出选择（未使用，保持默认 `0x00`）。
* **[★核心] CTRL_REG3_G (0x12)**: 陀螺仪高级/滤波配置。
  * `[7] LP_mode`: 低功耗模式使能 (0为常规模式)。
  * `[6] HP_EN`: 高通滤波器使能。
  * `[5:4] 保留`: 必须为 0。
  * `[3:0] HPCF_G[3:0]`: 高通滤波截止频率选择。在 ODR=119Hz 且设为 `0110` 时约为 0.1Hz。
  * **▶ 本架构配置**: `0x46` (开启高通滤波，消除静态零偏漂移)。
* **CTRL_REG4 (0x1E)**: 陀螺仪轴使能与路由。
  * `[7:6] 保留`: 0。
  * `[5] Zen_G`, `[4] Yen_G`, `[3] Xen_G`: 对应 Z, Y, X 轴使能 (默认均为 1)。
  * `[2] 保留`: 0。 `[1] LIR_XL1`: 锁存中断。 `[0] 4D_XL1`: 4D方向。
  * **▶ 本架构配置**: 保持默认 `0x38` (开启陀螺仪 X/Y/Z 轴)。

### 2.3 加速度计控制寄存器区 (Accelerometer Control)
* **CTRL_REG5_XL (0x1F)**: 加速度计轴使能。
  * `[7:6] DEC[1:0]`: 传感器数据更新降采样。
  * `[5] Zen_XL`, `[4] Yen_XL`, `[3] Xen_XL`: 加速度计轴使能 (默认均为 1)。
  * **▶ 本架构配置**: 保持默认 `0x38`。
* **[★核心] CTRL_REG6_XL (0x20)**: 加速度计基础配置。
  * `[7:5] ODR_XL[2:0]`: 采样率。`011`=119Hz。
  * `[4:3] FS_XL[1:0]`: 量程。`00`=±2g, `10`=±4g, `11`=±8g, `01`=±16g。
  * `[2] BW_SCAL_ODR`: 设为 0 使带宽自动由 ODR 决定。
  * `[1:0] BW_XL[1:0]`: 防混叠滤波带宽选择。
  * **▶ 本架构配置**: `0x70` (119Hz, ±4g, 自动带宽)。
* **[★核心] CTRL_REG7_XL (0x21)**: 加速度计高级/滤波配置。
  * `[7] HR`: 高分辨率模式使能。
  * `[6:5] DCF[1:0]`: 截止频率。`00`=ODR/9 (当 119Hz 时为 13.2Hz)。
  * `[4:3] 保留`: 0。
  * `[2] FDS`: 将滤波后的数据发送到输出寄存器与 FIFO。
  * `[1] 保留`: 0。 `[0] HPIS1`: 高通滤波中断。
  * **▶ 本架构配置**: `0xC4` (开启高分辨率，数字滤波 13.2Hz 存入输出)。

### 2.4 系统、总线与 FIFO 控制区
* **[★核心] CTRL_REG8 (0x22)**: 系统与总线控制。
  * `[7] BOOT`: 重启内存内容。
  * `[6] BDU`: Block Data Update 防数据撕裂机制 (0=连续更新, 1=等待高低字节均被读取后再更新)。
  * `[5] H_LACTIVE`: 中断引脚电平 (0=高电平有效, 1=低电平有效)。
  * `[4] PP_OD`: 中断引脚类型 (0=推挽, 1=开漏)。
  * `[3] SIM`: SPI 模式选择 (0=4线, 1=3线)。
  * `[2] IF_ADD_INC`: 寄存器地址自动递增使能 (突发读取必备)。
  * `[1] BLE`: 大小端选择 (0=小端, 1=大端)。
  * `[0] SW_RESET`: 软件复位 (置 1 触发，自动清零)。
  * **▶ 本架构配置**: 初始化首步写 `0x05` 软复位；随后写 `0x44` (开启 BDU 和 IF_ADD_INC)。
* **[★核心] CTRL_REG9 (0x23)**: FIFO 内存控制。
  * `[7] 保留`: 0。 `[6] SLEEP_G`: 陀螺仪休眠使能。 `[5] 保留`: 0。
  * `[4] FIFO_TEMP_EN`: 温度数据存入 FIFO 使能。
  * `[3] DRDY_mask_bit`: 数据就绪掩码。
  * `[2] I2C_DISABLE`: 禁用 I2C 总线接口。
  * `[1] FIFO_EN`: FIFO 内存使能。
  * `[0] STOP_ON_FTH`: 达到阈值停止收集。
  * **▶ 本架构配置**: `0x00` (彻底禁用 FIFO 及温度缓存，仅保留输出寄存器)。
* **[★核心] FIFO_CTRL (0x2E)**: FIFO 模式与阈值控制。
  * `[7:5] FMODE[2:0]`: FIFO 模式选择。`000`=Bypass 模式, `110`=Continuous 模式。
  * `[4:0] FTH[4:0]`: FIFO 深度阈值。
  * **▶ 本架构配置**: `0x00` (完全旁路 Bypass 模式)。

### 2.5 扩展与未使用寄存器 (供后续开发参考)
* `ACT_THS (0x04) / ACT_DUR (0x05)`: 运动唤醒阈值与时间配置。
* `INT_GEN_CFG_XL (0x06) ~ INT_GEN_DUR_XL (0x0A)`: 线性加速度中断生成配置（用于自由落体、撞击检测）。
* `ORIENT_CFG_G (0x13)`: 6D 方向识别配置。
* `OUT_TEMP_L/H (0x15-0x16)`: 芯片内部温度输出数据寄存器。
* `STATUS_REG (0x17 & 0x27)`: 包含陀螺仪/加速度计新数据就绪 (ZYXDA) 与覆盖 (ZYXOR) 标志。

## 3. 目标配置与初始化流程

依据上述寄存器映射，单片机开机时的 SPI 初始化序列应如下（需穿插适当延时）：
1. **通信链路验证 (WHO_AM_I)**: 
   * 读取 `WHO_AM_I (0x0F)` 寄存器。
   * 注意 SPI 读操作需将最高位置 1（即发送指令 `0x8F`）。
   * 验证返回值是否为 `0x68`。若不等于 `0x68`，说明 SPI 硬件连接或时序错误，应终止初始化并报错。
2. **系统软复位**: 写入 `CTRL_REG8 (0x22)` = `0x05`，延时 20ms。
3. **总线与防撕裂**: 写入 `CTRL_REG8 (0x22)` = `0x44` (BDU 开启防读取错位，地址自增开启)。
4. **关闭冗余机制**:
   * `CTRL_REG9 (0x23)` = `0x00` (禁用 FIFO)
   * `FIFO_CTRL (0x2E)` = `0x00` (Bypass 模式)
   * `INT1_CTRL (0x0C)` = `0x00` (禁用所有硬件中断)
5. **启动传感器**:
   * `CTRL_REG1_G (0x10)` = `0x68` (陀螺仪: 119Hz, ±500dps, 带宽14Hz)
   * `CTRL_REG3_G (0x12)` = `0x46` (陀螺仪: 开启高通滤波)
   * `CTRL_REG6_XL (0x20)` = `0x70` (加速度计: 119Hz, ±4g, 自动带宽)
   * `CTRL_REG7_XL (0x21)` = `0xC4` (加速度计: 高分辨率，数字滤波13.2Hz)

## 4. 主动查询与 DMA 搬运时序逻辑 (Active Polling & DMA)

由于禁用了硬件中断和 FIFO，数据由主控（STM32 的 10ms TIM2 定时器）强制发起周期性采集。

### 4.1 寄存器地址连续性与跳跃
LSM9DS1 具备特殊的智能地址跳跃功能：
连续读取起址设为陀螺仪 X 轴低字节 **`OUT_X_L_G (0x18)`**。硬件读完 6 字节陀螺仪数据至 `0x1D` 后，指针会自动跳过中间控制寄存器，直接跃迁至加速度计 X 轴低字节 **`OUT_X_L_XL (0x28)`**。
因此，只需一次连续读取 **12 个字节** 即可完美拼接 6 轴数据。

### 4.2 全双工 SPI DMA 操作流 (100Hz 触发)
1. **拉低片选**: 软件将 `PD7` (CS_A/G) 置为低电平。
2. **准备发送缓冲**: 提供长度 13 字节的数组 `tx_buf`。
   * `tx_buf[0] = 0x18 | 0x80 = 0x98` (读模式起址)。
   * `tx_buf[1] ~ tx_buf[12] = 0x00` (提供 12 个空字节提供 SPI 时钟以换取数据)。
3. **启动 DMA**: 调用 `HAL_SPI_TransmitReceive_DMA(&hspi1, tx_buf, rx_buf, 13)`。
4. **数据解析 (DMA TC 回调中)**:
   * **立刻拉高片选**: `PD7` 置为高电平，结束通信。
   * **剔除废位**: `rx_buf[0]` 为发指令期的回传杂波，直接舍弃。
   * **小端组装**: 将 `rx_buf[1]` 至 `rx_buf[12]` 按每两字节拼接为 6 个 `int16_t` 补码（依次为 Gyro X/Y/Z, Accel X/Y/Z）。
   * 直接将结果压入 Ring Buffer 的当前帧。
