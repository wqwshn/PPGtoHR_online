# MAX30101 PPG 传感器本项目配置说明

> 基于 `Core/Src/MAX30101.c`、`MAX30101_2.c`、`sample_rate_config.h` 和 `main.c` 实际配置整理。
> 详细的寄存器位域说明请参考 `docs/MAX30101寄存器配置说明.md` (数据手册总结)。

---

## 1. 硬件接口

本项目使用 **两片 MAX30101**, 分别通过独立的软件 I2C 总线连接:

| 通道 | 源文件 | I2C 总线 | 7-bit 地址 | 通信方式 |
|------|--------|---------|-----------|---------|
| PPG_CH1 | MAX30101.c/h | IIC1 (软件模拟) | 0x57 | 阻塞式 GPIO 位操作 |
| PPG_CH2 | MAX30101_2.c/h | IIC2 (软件模拟) | 0x57 | 阻塞式 GPIO 位操作 |

通道选择由 `main.h` 中 `PPG_DEFAULT_CHANNEL` 决定, 默认使用通道 2 (IIC2 总线)。

---

## 2. 工作模式概览

本项目支持两种 PPG 工作模式, 由 `main.h` 中 `CURRENT_WORK_MODE` 编译时选择:

| 模式 | 宏值 | LED 组合 | FIFO 每样本字节数 |
|------|------|---------|------------------|
| 心率模式 (默认) | MODE_HEART_RATE (0) | Green + Red + IR (Multi-LED) | 9 字节 |
| 血氧模式 | MODE_SPO2 (1) | Red + IR (SpO2) | 6 字节 |

---

## 3. 心率模式配置 (Multi-LED, 三光路)

### 3.1 初始化序列 (`MAX30101_Init` + `PPG_Config_Green_Hardcoded`)

**第一步: MAX30101_Init() 通用初始化**

```
0x00 = 0x00    // 清除中断状态1
0x01 = 0x00    // 清除中断状态2
0x02 = 0x00    // 关闭所有中断
0x03 = 0x00    // 关闭温度中断
0x04 = 0x00    // FIFO_WR_PTR 清零
0x05 = 0x00    // OVF_COUNTER 清零
0x06 = 0x00    // FIFO_RD_PTR 清零
0x08 = 0x10    // FIFO: 无平均 + ROLLOVER_EN + A_FULL=0
0x09 = 0xC7    // MODE: SHDN + RESET + Multi-LED (复位并启动)
0x0A = 0x7F    // SPO2: ADC=16384nA + SR=3200sps + PW=411us/18bit
0x0C = 0xB1    // Red LED 电流 ~35mA
0x0D = 0xB1    // IR LED 电流 ~35mA
0x0E = 0xB1    // Green LED 电流 ~35mA
0x11 = 0x01    // SLOT1=Red, SLOT2=Disabled
0x12 = 0x00    // SLOT3/4=Disabled
```

**第二步: PPG_Config_Green_Hardcoded() 覆写为三光路配置**

```
0x09 = 0x07    // MODE: Multi-LED (不复位, 不休眠)
0x11 = 0x13    // SLOT1=Green(011), SLOT2=Red(001)
0x12 = 0x02    // SLOT3=IR(010), SLOT4=Disabled
0x0E = 0x3F    // Green LED 电流 ~12.6mA
0x0C = 0x3F    // Red LED 电流 ~12.6mA
0x0D = 0x3F    // IR LED 电流 ~12.6mA
0x0A = 由采样率决定 (见下表)
0x08 = 由采样率决定 (见下表)
0x04~06 = 0x00 // 再次清空 FIFO 指针
```

### 3.2 采样率相关寄存器 (sample_rate_config.h)

本项目固定 `PPG_SAMPLE_RATE = 100`, 对应参数:

| 寄存器 | 地址 | 值 | 说明 |
|--------|------|-----|------|
| SPO2_CONFIG | 0x0A | **0x52** | ADC=8192nA, SR=800sps, PW=215us (17-bit) |
| FIFO_CONFIG | 0x08 | **0x7F** | SMP_AVE=8x, ROLLOVER_EN, A_FULL=15 |

**采样率关系**: 内部 ADC 以 800sps 采样, 8x 硬件平均后有效输出 100sps。

**三光路时序约束**: 3 LED x 215us = 645us < 1250us (1/800sps), 满足时序要求。
因此 100Hz 三光路模式下必须降至 17-bit (215us), 不能使用 18-bit (411us)。

### 3.3 各采样率参数对比

| 采样率 | TIM16 ARR | SPO2_CONFIG | FIFO_CONFIG | ADC内部 | 平均 | 脉宽 |
|--------|-----------|-------------|-------------|---------|------|------|
| 50Hz | 1999 | 0x73 | 0x9F | 800sps | 16x | 411us (18-bit) |
| **100Hz** | **999** | **0x52** | **0x7F** | **800sps** | **8x** | **215us (17-bit)** |
| 125Hz | 799 | 0x77 | 0x5F | 1000sps | 4x | 411us (18-bit) |

> 当前仅 100Hz 模式支持三光路 (G+R+IR), 其余采样率尚未适配多光路时隙。

## 4. 血氧模式配置 (SpO2, Red+IR)

当 `CURRENT_WORK_MODE = MODE_SPO2` 时, 由 `PPG_Config_SpO2_Hardcoded()` 配置:

```
0x09 = 0x03    // MODE: SpO2 (Red + IR)
0x0C = 0x91    // Red LED 电流 ~29mA
0x0D = 0x91    // IR LED 电流 ~29mA
0x0A = 0x6F    // ADC=16384nA + SR=400sps + PW=411us (18-bit)
0x08 = 0x1F    // 无平均 + ROLLOVER_EN + A_FULL=15
0x04~06 = 0x00 // 清空 FIFO 指针
```

### 4.1 与心率模式差异

| 项目 | 心率模式 | 血氧模式 |
|------|---------|---------|
| MODE 寄存器 | 0x07 (Multi-LED) | 0x03 (SpO2) |
| 活跃 LED | Green + Red + IR | Red + IR |
| LED 电流 | 0x3F (12.6mA) | 0x91 (~29mA) |
| ADC 量程 | 8192nA | 16384nA |
| 内部采样率 | 800sps | 400sps |
| 硬件平均 | 8x | 无 |
| ADC 分辨率 | 17-bit | 18-bit |
| FIFO 每样本 | 9 字节 | 6 字节 |
| 温度补偿 | 无 | 1Hz 非阻塞测温 |

## 5. FIFO 数据读取

### 5.1 读取流程 (主循环, 每次 ADC 完成后)

```
1. 读取 FIFO_WR_PTR (0x04) 和 FIFO_RD_PTR (0x06)
2. 计算可用样本数: sample_count = (wr_ptr - rd_ptr) & 0x1F
3. 循环读取 sample_count 个样本:
   - 心率模式: 每样本 9 字节 (Green 3B + Red 3B + IR 3B)
   - 血氧模式: 每样本 6 字节 (Red 3B + IR 3B)
4. 多样本取均值, 缓存上次有效值防止空读跳变
```

### 5.2 数据解包 (17-bit, 心率模式)

```c
// 24-bit 左对齐原始数据, 右移 1 位得到 17-bit 有效值
uint32_t raw = (buf[0] << 16) | (buf[1] << 8) | buf[2];
uint32_t value = (raw >> 1) & 0x01FFFF;
```

### 5.3 数据解包 (18-bit, 血氧模式)

```c
// 24-bit 左对齐原始数据, 直接掩码 18-bit
uint32_t value = (buf[0] << 16 | buf[1] << 8 | buf[2]) & 0x03FFFF;
```

## 6. 数据协议映射

### 6.1 心率模式 (33 字节原始数据包)

PPG 数据位于 `allData[PPG_START_INDEX]` 起始的 9 字节:

| 偏移 | 字段 | 说明 |
|------|------|------|
| 22~24 | PPG Green | 17-bit 均值, 高位在前 |
| 25~27 | PPG Red | 17-bit 均值, 高位在前 |
| 28~30 | PPG IR | 17-bit 均值, 高位在前 |

### 6.2 血氧模式

PPG 数据借用前 3 字节存储温度, 后 6 字节存储 Red/IR:

| 偏移 | 字段 | 说明 |
|------|------|------|
| 22 | 温度整数 | 芯片内部温度 (degC) |
| 23 | 温度小数 | 精度 0.0625 degC |
| 24 | 保留 | 0x00 |
| 25~27 | PPG Red | 16-bit 均值 (>>2), 高位在前 |
| 28~30 | PPG IR | 16-bit 均值 (>>2), 高位在前 |

## 7. PPG 通道选择机制

`ppg_channel.c/h` 封装了通道选择, 所有操作通过 `PPG_` 前缀统一调用:

```c
PPG_SetChannel(PPG_CH2);      // 切换到通道2
PPG_Init();                    // 初始化当前通道的 MAX30101
PPG_ReadOneByte(0xFF);         // 读当前通道寄存器
PPG_WriteOneByte(0x09, 0x07);  // 写当前通道寄存器
PPG_ReadFIFO_Burst(buf, 9);   // 批量读当前通道 FIFO
```

## 8. 寄存器速查表 (心率模式 100Hz)

| 地址 | 名称 | Init值 | Config覆写值 | 说明 |
|------|------|--------|-------------|------|
| 0x00~0x03 | 中断 | 全0 | - | 不使用中断 |
| 0x04~0x06 | FIFO指针 | 0x00 | 0x00 | 每次配置后清零 |
| 0x07 | FIFO_DATA | 只读 | - | 批量读取窗口 |
| 0x08 | FIFO_CONFIG | 0x10 | **0x7F** | 8x平均 + 回滚 |
| 0x09 | MODE_CONFIG | 0xC7 | **0x07** | Multi-LED |
| 0x0A | SPO2_CONFIG | 0x7F | **0x52** | 8192nA + 800sps + 17bit |
| 0x0C | LED1_PA (Red) | 0xB1 | **0x3F** | 12.6mA |
| 0x0D | LED2_PA (IR) | 0xB1 | **0x3F** | 12.6mA |
| 0x0E | LED3_PA (Green) | 0xB1 | **0x3F** | 12.6mA |
| 0x11 | MULTI_LED_CTRL1 | 0x01 | **0x13** | SLOT1=Green, SLOT2=Red |
| 0x12 | MULTI_LED_CTRL2 | 0x00 | **0x02** | SLOT3=IR, SLOT4=Off |
| 0xFF | PART_ID | 只读 | - | 期望值 0x15 |
